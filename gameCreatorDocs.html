Text-Based Adventure Game Creator Documentation

The game creation tool is designed to be somewhat user-friendly, but not to protect the user from themselves in every possible scenario.
Mainly, there are the following things you need to look out for or the game player may throw an error:

  • Unless the "Option goes to other screen" or "Is end screen" option is checked, you must have an option in the options list.
  • The "purpose" and "text" attributes should be set in each option.
  • Do not have more than one option with the same name, as it will just be processed as the last value that was unique.

How to Use The Tool

Set the name of your game in the top-left corner. You can edit values in all the text boxes.
Create a new option by pressing the "+" button. The game file follows a nested structure of options.
For an end screen, check the "Is end screen" checkbox. "
Check "Option goes to other screen" to jump around to a seperate screen and then set the "goto" textbox to a path.
A path references a place on the tree. For example, the path for the screen for door 1 would be "screens.options.door 1".
This is not recommended, but you can disable the user's ability to type "go back" to undo the last choice by checking the "Disable undoing previous choice" checkbox.

Notes On Unintended Behaviors of The Tool

When you toggle a checkbox or add/remove an option, the open/closed state of the dropdowns will reset.
When you change the name of an option, it will be listed last in the game file and in game in the screen it is in. This can be remedied by renaming and naming back all other options in the screen.
When you press space when renaming an option, the open/closed state of the dropdown will be toggled.
